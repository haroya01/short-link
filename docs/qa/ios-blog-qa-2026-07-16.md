# kurl 블로그 iOS 전수 QA + 벤치마크 — 2026-07-16

> 대상: kurl-ios main 1ea05ad (#188 포함). 방식: 표면별 5트랙 — 기존 XCUITest 회귀망 전량 + 표면별 탐사 트래버스(신규 작성, 모든 버튼 탭→실상태변화 단언) + 스크린샷 증거(라이트/다크 총 ~250장, scratchpad qa-*/ 폴더) + 트위터·스레드·Medium 대비 벤치마크.
> 실행 환경: Xcode 27 beta, iOS 27.0 시뮬 4대(에이전트당 1대·슬롯 승인제 — 7대 동시 부팅 시 testmanagerd 포화 실측, 3~4대가 안전선).

## 총평

**"존재하는 모든 흐름과 버튼" 판정: 사용자 대면 데드 버튼·죽은 항해 0.** 5표면 전수 트래버스에서 눌러서 아무 일도 안 일어나는 컨트롤은 하나도 없었다. 과거 함정 4종(유리 contentShape 데드존·값형 NavLink 항해 죽음·item 시트 재발화·confirmationDialog 중앙 팝오버) 전부 무회귀.

**실결함 6건 발견 — 전부 수정 착수(4 PR 트랙).** 기능이 "죽은" 것은 노트 컴포저 단 하나(탭바 가림)였고, 나머지는 도달 불가 상태(설정 하단·검색 무결과)와 파리티/폴리시 갭.

**벤치마크: 축 다수에서 동급~상회, 미달은 국소.** 미달점은 전부 "쓰는 순간의 마이크로 피드백"과 상태화면 마감에 몰려 있고, 아키텍처 변경 없는 폴리시로 회수 가능 — 이번 수정 라운드에 전부 포함됨.

## 3. 벤치마크 판정 (트위터/스레드/Medium 대비)

### 상회
- **게스트 경험**: 공개 읽기·검색 풀 오픈(그들은 로그인 벽 강화 추세), 인증 액션엔 가치 프레이밍 로그인 시트, 웰컴이 제품 서사를 시연(형광→실→컬렉션)
- **분석 화면**: 기간칩·area 차트·수명 스탯·정렬칩·유입경로 — 트위터 애널리틱스/Medium 스탯보다 밀도·완성도 높음
- **상태 3분리 규율**: 로딩=스켈레톤(스피너 0)·빈=CTA·에러=재시도, 위장 0 — 그들이 자주 뭉개는 지점
- **네이티브 초안 프리뷰**: 발행 후 모습을 네이티브 타이포로 — 스레드엔 없음, Medium급 리딩
- **롱프레스 컨텍스트 메뉴**: preview lift+고유 동사(컬렉션 연결)까지 — 동급 이상

### 동급
- 다크 모드(균질·고대비·차트까지), 타이포 위계·간격 리듬, 카드 press/햅틱/pill 활주 모션, 행 리듬, 설정 화면 관습(탭바 건 제외), 헤딩 active 배지(오히려 Medium보다 명확)

### 미달 → 전부 이번 라운드 수정
- 쓰기 마이크로 피드백: 서식 적용 순간 raw 마커 노출(B1)·툴바 햅틱 0(B2)·발행 축하에 키보드 잔존(B3)·프레스 상태 부재(B4)·자동저장 스피너 모호(B5)
- 에러 상태가 stock iOS로 후퇴(빈 상태와 격차) — 브랜드화 진행
- 탭바 가림 3종(§2)

### 다른 선택 (미달 아님 — 의도)
- 목록 썸네일 없는 글행 카탈로그(스캔성 vs 조용함), 정보 밀도 여유(글 중심), 배너 없는 프로필, 그린 단일 액센트

## 4. 사용자 결정 대기 (수정 안 함)

1. **커버 카드 트리트먼트**: 스크린샷 원본 커버가 피드 소음(Medium은 크롭+옅은 오버레이) — 카드 문법 변경이라 취향 결정
2. **스플래시 1.15s 매 실행**: 웰컴 연출 의도 vs 재방문자 첫 진입 지연 — 정책 결정
3. **게스트 매 실행 웰컴 커튼(#170)**: 로그인 유도 vs 반복 마찰 — 정책 결정 ("둘러보기" 출구가 있어 완화됨)
4. **앱내 테마 선택 부재**: 시스템 외형만 따름(그들은 앱내 선택 제공) — 기능 추가 결정
5. **표 정렬 캐럿 열 감지**: Phase 2b 설계 보류 존중 — 재개 여부
6. **V2 undo/redo**: 엔진 히스토리 인프라 조사 결과에 따라 (조사 중)

## 5. 테스트 위생 백로그 (앱 수정 아님)

- 기존 UITest 픽스처를 실서버 핀 → 목 데이터 핀으로 이관 (`/public/*` 목 커버리지 확장 포함: 피드·시리즈·발견 하이라이트·프로필 posts)
- 발견 탭 재설계(카드피드→입구) 이전 가정의 테스트 2건 갱신
- 이중 해시 태그(##…) 데이터 정규화 — 백엔드 태그 저장 정규화 확인
- ci/ios-uitests 브랜치(informational 레인) 머지 여부 — GH 러너 Xcode 27 부재라 당분간 빨간불 표시만
- kurl-ios SourcePackages 2경로 gitignore 누락
- **WriteV2CanvasFocusUITests.testTapEmptyCanvasStartsWriting = 테스트 격리 defect(앱 무죄).** 이전 컴포즈 테스트(ComposeRecovery 강제종료·트래버스 이탈보호)가 ComposeRecoveryStore 파일 스태시를 남기면 다음 새-컴포즈에서 복구 다이얼로그("저장되지 못한 본문이 있어요")가 캔버스를 덮어 활주로 탭+typeText가 "no keyboard focus"로 실패. 스태시 있으면 5/5 실패·erase+웜이면 통과(12s)로 증명. 복구 기능은 정상. 수정=setUp에서 스태시 리셋(예: `--reset-recovery` 런치 플래그로 ComposeRecoveryStore 와이프) 또는 다이얼로그 선-소거
- **ComposeStressUITests.testPastingThreeImagesStacksThreeImageBlocks = 취약 셀렉터 defect(앱 무죄).** pasteViaMenu가 `app.textViews.firstMatch`를 롱프레스 → 이미지 블록 삽입 후 firstMatch가 {{inf,inf},{0,0}} 퇴화 프레임 TextView로 잡혀 kAXScrollToVisibleAction 실패(3/3 재현, 경합 무관). 다중이미지 붙여넣기 기능 자체는 정상(RealInteraction.testPasteClipboardImageBecomesImageBlock 통과). 수정=firstMatch 대신 구체적 가시 TextView(boundBy 또는 "본문 이어 쓰기" 활주로) 지정
